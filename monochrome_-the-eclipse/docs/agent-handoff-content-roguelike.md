# Agent Handoff: Content and Roguelike Loop

이 문서는 다음 작업자가 Google Drive를 매번 다시 뒤지지 않고 현재 구현 상태를 이어받기 위한 요약이다.

## 기준 문서

- `탐험 시스템 설계서`: 15턴 x 3스테이지, 매턴 3개 선택지, 미니보스 5~6턴, 보스 15턴, 이벤트 중복 방지.
- `코인 전투 시스템`: 5개 동전, 족보 선택, 능동 스킬 개입, 전투 후 보조/족보 성장 선택.
- `자원 및 상점 시스템 컨셉`: 에코/감각/기억 자원, 상점 소비, 감각/성장/메타 반복 구조.
- `몬스터 컨셉`: Stage 2 몬스터 3종, 미니보스 1종, 보스 1종의 수치/스킬/패시브.

## 현재 구현 상태

- Stage 1은 기존 루프 유지.
- Stage 2는 문서 기반 몬스터/패턴/패시브가 들어가며 플레이 가능.
- Stage 3은 문서상 테마만 있고 상세 몬스터/이벤트/보스가 부족해서 잠금 상태다. 추후 대체/변경 필요.
- 전투 보상은 자원 보상 + 기존 `playerSkillUnlocks` 기반 기술 드래프트를 제공한다. 경험치/레벨업 곡선이 없어서 임시 직접 드래프트다. 추후 대체/변경 필요.
- 엘리트/보스 보상은 `dataUpgrades.ts`의 미보유 패시브 드래프트도 제공한다. 문서의 패시브 성장 방향을 직접 반영한 부분이다.
- 신규 런은 문서 기준대로 행운 동전 1개를 들고 시작하며, 최대 보유량은 7개다. 내부 변수명은 기존 `reserveCoins`를 유지한다.
- Stage 2 전투/탐험 BGM은 실제 파일 대신 절차적 프리셋으로 분리되어 있다. 최종 음원 파일 확보 시 `utils/audioManifest.ts` 키를 유지하고 교체한다.
- Stage 2 몬스터 5종은 생성 에셋이 연결되어 있다. `shadowWraith`, `doppelganger`, `unpleasantCube`, `subject162`, `chimera`가 `public/assets/monsters` 아래 파일을 참조한다.
- 키메라는 HP 50%/30% 이하에서 페이즈 라벨과 패턴셋이 바뀐다.
- 상점 패시브 60개 중 43개는 전투 엔진 또는 보상 처리에서 참조된다.
- `RESONANCE`는 2턴 카운트다운 후 누적 수치만큼 방어 무시 고정 피해로 발동한다.

## 계속 봐야 할 항목

- `공명` 계열 잔여 패시브: 지속 시간 연장, 반동, 보호막, 자기혐오는 정확한 타이밍 정의가 더 필요하다.
- 증폭 상한/증폭 보너스 패시브: 기본 상한 10, 확장 상한 20, 방어 동시 획득 규칙을 한 번에 정리해야 한다.
- Stage 3: 상세 몬스터/보스/이벤트 문서가 없으므로 임의 생성 금지.
- 보상 곡선: 현재 전투마다 기술 드래프트가 가능하므로, 추후 문서에 경험치/레벨업 빈도가 확정되면 대체해야 한다.
- 저주 거래: Drive 검색에서 구체 규칙이 나오지 않아 구현 보류. 추후 문서가 생기면 이벤트/상점 보상 테이블로 연결한다.
- Stage 2 전용 이벤트/배경 파일: 문서상 테마는 있으나 이벤트 본문/배경 파일 상세가 부족해서 현재는 공통 이벤트와 절차적 전투 분위기만 사용한다.
