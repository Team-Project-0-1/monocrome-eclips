# 출시 방향 및 품질 기준

마지막 업데이트: 2026-05-01

## 결론

현재 방향은 **유료 1.0 상업 출시가 아니라 무료 공개 프로토타입 / 버티컬 슬라이스**가 맞다.

판단 근거:
- Drive 기획 문서는 전체 구조를 3스테이지, 스테이지당 15턴, 총 45턴으로 잡고 있다.
- 현재 빌드는 핵심 동전 전투 루프와 1-2스테이지 플레이 검증에는 쓸 수 있지만, 3스테이지는 아직 세부 기획과 구현이 잠겨 있다.
- 이 상태를 완성 상업 게임으로 팔면 콘텐츠 완결성, 기대 관리, 환불 리스크가 모두 맞지 않는다.

## 공개 프로토타입 기준

목표: 돈을 받는 것이 아니라 핵심 재미 검증, 피드백 수집, 초기 팬층 확인.

최소 기준:
- 첫 5분 안에 동전 앞/뒤, 패턴 선택, 피해 예측의 의미가 이해된다.
- 짧은 런 하나가 크래시 없이 끝난다.
- 최소 1개 캐릭터가 확실한 플레이 감각을 가진다.
- `npm run check`와 보안 감사가 통과한다.
- 페이지와 홍보 문구가 prototype, demo, work-in-progress 중 하나로 명확히 표시된다.
- 3스테이지와 엔딩 완성을 약속하는 식의 유료 상품처럼 보이지 않는다.

현재 상태:
- 기술 게이트는 통과했다.
- 무료 공개 프로토타입 기준에는 들어온다.
- 3스테이지 미완성을 정직하게 표기해야 한다.

권장 배포:
- 무료 웹 프로토타입.
- itch 스타일 공개 페이지 또는 devlog와 함께 공개.
- 피드백 폼, 알려진 미구현 범위, 다음 로드맵을 같이 둔다.

## 유료 얼리 액세스 기준

목표: 미완성임을 공개하되, 현재 빌드만으로도 돈을 낸 사용자가 납득할 수 있는 가치 제공.

최소 기준:
- 1-2스테이지가 반복 플레이 가능한 수준으로 다듬어져 있다.
- 3스테이지는 구현 전이라도 PRD, 몬스터/보스/이벤트 표, 보상 압박 구조가 문서로 잠겨 있다.
- 최소 2개 이상 캐릭터가 재플레이 이유를 만든다.
- 5회 이상 런을 돌릴 이유가 있다.
- 누락 콘텐츠가 결제 전에 명확히 고지된다.
- 이미지/오디오/타이틀의 상업 사용 권리가 확인되어 있다.
- 업데이트 주기와 우선순위가 공개 가능하다.

현재 상태:
- 아직 권장하지 않는다.
- 3스테이지 기획 확정, 외부 플레이테스트, 밸런스 데이터, 에셋 권리 검토가 선행되어야 한다.

## 유료 1.0 상업 출시 기준

목표: 변명 없이 하나의 완성 게임으로 판매.

최소 기준:
- 계획된 3스테이지가 모두 설계, 구현, 밸런싱, 클리어 검증을 마쳤다.
- 각 스테이지에 고유한 몬스터, 보스 정체성, 이벤트 풀, 시청각 톤, 보상 압박이 있다.
- 전체 런에 실제 엔딩과 승리 화면이 있다.
- 4개 캐릭터가 서로 다른 빌드와 해금 동기를 가진다.
- 몇 시간 이상 반복해도 발견할 조합과 목표가 남아 있다.
- 온보딩, 툴팁, 전투 예측, 결과 피드백이 개발자 설명 없이 이해된다.
- 임시 생성 에셋의 상업 사용 리스크가 해소되어 있다.
- 메뉴, 캐릭터 선택, 탐험, 전투, 보상, 상점, 휴식, 이벤트, 스테이지 클리어, 패배, 승리를 자동 회귀 테스트가 커버한다.
- 외부 테스터가 왜 이겼고 왜 졌는지 설명할 수 있다.

현재 상태:
- 준비되지 않았다.
- 가장 큰 문제는 빌드 품질보다 콘텐츠 완결성과 재미/반복성의 증거 부족이다.

## 권장 의사결정

1. 첫 공개 버전은 **무료 공개 프로토타입**으로 라벨링한다.
2. 핵심 검증 질문은 “동전 패턴 전투가 플레이어를 다시 오게 만드는가”로 둔다.
3. 3스테이지 기획이 잠기고 첫 30분 플레이 피드백이 좋아지기 전에는 유료 판매하지 않는다.

## 다음 게이트

공개 프로토타입 전:
- 릴리스 라벨을 `Prototype v0.1` 또는 동급으로 확정한다.
- 3스테이지 잠금/미구현 문구를 공개 페이지와 게임 내 흐름에 맞춘다.
- 에셋/오디오 권리 검토를 최소한 프로토타입 공개 범위에서 승인한다.
- 공개 URL과 호스팅 방식을 정한다.

유료 얼리 액세스 전:
- 3스테이지 PRD가 존재한다.
- 3스테이지 몬스터, 보스, 이벤트 테이블이 존재한다.
- 외부 플레이테스트 리포트가 최소 10건 있다.
- 최소 2개 캐릭터가 테스터에게 “다시 해볼 이유가 있다”는 평가를 받는다.
- 첫 런 이탈 지점과 전투 패배 원인을 기록할 수 있다.
- 상업 사용 에셋 권리가 승인되어 있다.

유료 1.0 전:
- 3스테이지와 승리 루트가 구현되어 있다.
- 전체 런 브라우저 E2E가 있다.
- 모든 캐릭터 밸런스 패스가 끝났다.
- 상점/공개 페이지 문구가 실제 콘텐츠와 일치한다.
- 환불 리스크가 큰 미구현/오해 요소가 닫혀 있다.

## 즉시 우선순위

1. 3스테이지를 구현하지 말고 먼저 설계 문서를 잠근다.
2. 현재 빌드는 무료 프로토타입으로 공개할지 사용자 승인을 받는다.
3. 브라우저 스모크 테스트를 코드화해서 매 빌드 반복 검증한다.
4. 런 결과, 사망 원인, 캐릭터별 승률을 기록할 수 있는 가벼운 텔레메트리 또는 로컬 요약을 추가한다.
5. 외부 플레이테스트를 통해 유료 전환 판단 자료를 모은다.
